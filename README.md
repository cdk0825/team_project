# 🤖 AI 기반 서비스 QA 자동화 프로젝트

## 🌟 프로젝트 개요

이 프로젝트는 **QA(품질 보증) 업무의 효율성 극대화 및 품질 향상**을 목표로, **지정된 웹사이트를 대상**으로 한 테스트 자동화 실습 과정입니다. 참가팀은 실제 웹사이트의 기능과 품질 요소를 분석하고, **Selenium** 및 **Pytest**를 활용하여 테스트 코드를 작성합니다. 나아가 **Jenkins**를 이용한 CI/CD 파이프라인 구축을 통해 **반복 업무를 감소**시키고 **테스트의 신뢰성**을 확보하는 전 과정을 경험합니다.

---

## 🎯 프로젝트 목표

본 프로젝트를 통해 달성하고자 하는 핵심 목표는 다음과 같습니다.

1.  **효율성 증진:** 반복적인 수동 테스트를 자동화하여 테스트 수행 시간을 단축하고 효율성을 극대화합니다.
2.  **결함 발견 및 품질 향상:** 심층적인 자동화 테스트를 통해 추가적인 결함을 발견하고, 제품의 **품질 및 신뢰성**을 향상시킵니다.
3.  **CI/CD 환경 구축 경험:** Jenkins를 활용하여 지속적 통합/지속적 배포(CI/CD) 환경을 구축하고 자동화 파이프라인 운영 능력을 습득합니다.
4.  **반복 업무 최소화:** 자동화된 테스트를 통해 QA 과정에서 발생하는 반복 작업을 최소화합니다.

---

## ⛳ 프로젝트 주요 단계 및 기술 스택

이 프로젝트는 다음의 6단계 워크플로우를 따라 진행되며, 테스트 자동화를 위한 핵심 기술을 활용합니다.

### 1. 테스트 시나리오 작성

- **목표:** 반복적이고 중요한 핵심 기능을 중심으로 **자동화할 테스트 시나리오**를 선정하고 정의합니다.

### 2. 테스트 코드 작성 (Selenium & Pytest 활용)

- **기술 스택:** **Selenium** (웹 자동화 드라이버), **Pytest** (테스트 프레임워크)
- **내용:** 준비된 테스트 시나리오를 Python 코드로 변환하여 자동화 테스트 케이스를 구현합니다.

### 3. CI/CD 자동화 (Jenkins 활용)

- **기술 스택:** **Jenkins** (CI/CD 자동화 서버)
- **내용:** 코드가 변경되거나 정해진 시간마다 자동으로 테스트가 실행되고 그 결과를 확인할 수 있는 CI/CD 파이프라인을 구축합니다.

### 4. 테스트 실행 및 보고

- **내용:** 로컬 환경 또는 Jenkins 파이프라인을 통해 테스트를 실행하고, 발생한 실패 및 오류를 추적하여 결과를 확인합니다.

### 5. 문제 해결 및 최적화

- **내용:** 테스트 과정에서 발견된 문제의 원인을 분석하고 해결하며, 테스트 코드를 최적화하여 안정성과 실행 속도를 개선합니다.

### 6. 결과 보고 및 문서화

- **내용:** 최종 테스트 결과, 발견된 결함, 자동화 범위 및 효율성 증대 효과 등을 정리하여 최종 문서를 작성하고 보고합니다.

## 🚀 시작 가이드 (Getting Started)

프로젝트에 참여하기 위해 필요한 기본 환경 설정 및 절차는 다음과 같습니다.

1.  **환경 설정:** Python 설치 및 가상 환경 설정
    ```bash
    # python version : 3.11.9
    venv.bat # 가상환경 활성화
    ```
2.  **의존성 설치:** `requirements.txt` 파일을 참고하여 필요한 라이브러리 (Selenium, Pytest 등)를 설치합니다.
    ```bash
    pip install -r requirements.txt
    ```
3.  **코드 구현:** `2. 테스트 코드 작성` 단계에 따라 테스트 스크립트를 구현합니다.
4.  **Jenkins 연동:** Jenkins 서버에 파이프라인을 설정하고 자동화 빌드를 실행합니다.
